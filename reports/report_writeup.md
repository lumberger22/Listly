# Reports

## How the System Generates and Presents Reports

Listly includes a dedicated Reports page accessible from the main navigation bar. Reports are generated by the backend Express API, which queries the MySQL database using multi-table JOIN and GROUP BY aggregation queries. The frontend fetches both reports simultaneously on page load using Axios and renders them as styled tables on a single page. Reports are publicly accessible — no login is required — so marketplace activity is visible to all users.

---

## Report 1 — Seller Performance Summary

### What Data Is Shown

This report aggregates data across the User, Seller, Item_Listing, Transaction, and Review tables to produce one summary row per seller. For each seller it displays:

- **Active Listings** — the number of items they currently have available
- **Sold Listings** — the number of items they have successfully sold
- **Transactions** — the total number of completed sales
- **Total Revenue** — the sum of all completed transaction prices in dollars
- **Avg Rating** — their mean star rating calculated from all buyer reviews they have received
- **Reviews** — the total number of reviews written about them

Results are sorted by Total Revenue descending, so the highest-earning sellers appear at the top.

The SQL query uses four LEFT JOINs to connect the seller to their listings, transactions, and reviews, and uses GROUP BY on the seller's UID to collapse all related rows into a single summary. Aggregation functions used include COUNT(DISTINCT ...) for listings, transactions, and reviews, SUM with a conditional expression for active vs. sold counts, COALESCE(SUM(...), 0) to calculate revenue only from completed transactions while defaulting to zero for sellers with none, and AVG for the star rating.

### How It Is Useful to Users

**For buyers:** Before making an offer, a buyer can check this report to assess a seller's track record. A seller with many completed transactions and a high average rating signals that they are reliable and experienced. A seller with many active listings but zero sales may warrant more caution.

**For sellers:** This report lets sellers benchmark their own performance against others on the platform. Seeing total revenue and transaction counts provides context for how active the marketplace is and where they stand relative to peers.

## Report 2 — Category Market Overview

### What Data Is Shown

This report aggregates data across the Item_Listing and Offer tables to produce one summary row per category. For each category it displays:

- **Active** — the number of listings currently available in that category
- **Sold** — the number of listings that have already been purchased
- **Avg Price** — the average asking price of currently active listings
- **Min Price** — the lowest asking price among active listings
- **Max Price** — the highest asking price among active listings
- **Total Offers** — the total number of buyer offers ever placed on listings in that category

Results are sorted by Active Listings descending, so the most populated categories appear first.

The SQL query uses a LEFT JOIN between Item_Listing and Offer so that categories with no offers are still included. GROUP BY on Category collapses all listings into one row per category. Aggregation functions used include COUNT for total listings and offers, SUM with conditional expressions for active vs. sold counts, and AVG, MIN, and MAX with CASE WHEN expressions so that price statistics are computed only over active listings and not sold or removed ones.

### How It Is Useful to Users

**For buyers:** The price range columns (Avg, Min, Max) let a buyer quickly understand what to expect to pay in a given category before browsing individual listings. A large spread between Min and Max signals that deals are available if a buyer shops carefully.

**For sellers:** The Total Offers column acts as a demand signal. A category with many offers relative to its active listing count indicates strong buyer interest, which is a good signal for what to post next. The average price helps sellers set competitive asking prices without underpricing themselves.
